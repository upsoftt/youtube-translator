# PRD: YouTube Translator — Синхронный перевод YouTube-видео

## Видение продукта
Кроссплатформенное приложение, которое воспроизводит YouTube-видео с синхронным голосовым переводом в реальном времени. Пользователь вставляет ссылку — видео начинает играть, а поверх оригинальной звуковой дорожки звучит перевод на выбранном языке, синхронизированный с речью спикера.

## Целевая аудитория
- Русскоязычные зрители англоязычного контента YouTube
- Люди, предпочитающие аудио-перевод субтитрам
- Студенты, изучающие контент на иностранных языках

## Основные функции (MVP)
1. **Воспроизведение видео** через YouTube IFrame API (web) / expo-av (native)
2. **Извлечение текста**: субтитры YouTube (приоритет) → DeepGram STT (fallback)
3. **Перевод текста**: OpenAI GPT-4o-mini
4. **Голосовая озвучка**: DeepGram Aura-2 (en/es/de/fr) → Edge TTS (ru и др.)
5. **Синхронизация**: аудио-перевод привязан к таймкодам оригинала
6. **Управление**: громкость оригинала/перевода, seek с перезапуском pipeline, скорость воспроизведения

## Ключевые метрики
- **Задержка старта**: < 5 секунд от включения перевода до первого озвученного сегмента
- **Рассинхронизация**: < 1 секунды между позицией видео и озвученным переводом
- **Покрытие**: > 95% речи в видео переведено и озвучено

## Технические ограничения
- YouTube IFrame API не даёт доступ к аудио-дорожке напрямую — стримим отдельно через yt-dlp + ffmpeg
- DeepGram Aura-2 не поддерживает русский → fallback на Edge TTS (Python subprocess, +1-2с на сегмент)
- ffmpeg без `-re` сливает весь аудио-поток мгновенно → DeepGram получает весь файл, а не realtime chunks
- expo-av Audio.Sound не гарантирует мгновенный playback — есть задержка на декодирование base64

## Текущие критические проблемы (из аудита)
1. Polling-based sync (200мс + 500мс) вместо event-driven
2. Edge TTS: Python subprocess на каждый сегмент = 1-2с латентности
3. Sequential subtitle processing (await в цикле)
4. Нет timeout на resume_video → deadlock видео
5. Duration субтитров считается через gaps
6. ffmpeg без `-re` заканчивается мгновенно → «Стрим завершён» раньше обработки
